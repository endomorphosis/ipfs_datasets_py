## Modeled Scope

`XamanTestnetPayload.tla` models a finite two-client interleaving over one Testnet payload. It covers client-visible categorical resolution behavior:

- fetch, review, and approval before resolution;
- a single resolution lock before signing;
- signed, declined, expired, cancelled, duplicate-blocked, and replay-blocked terminal states;
- at most one submitted signed resolution;
- Testnet binding before review/sign/submit phases;
- redaction-preserving categorical state only.

The checked invariants are `TypeOK`, `AtMostOneSubmittedResolution`, `ResolvedPayloadIsTerminal`, `TerminalStateHasResolutionKind`, `NoReviewWithoutTestnetBinding`, `ResolutionRequiresPriorReview`, and `DuplicateClientCannotResolveAfterTerminal`.

## Current Result

Apalache `0.58.3` checked all seven invariants with `--no-deadlock`, because terminal payload states intentionally stop. No counterexample was found for the finite concurrency model.

The overall verdict is still blocked:

- `overall_status: checked_with_unresolved_threat_model_gaps`
- `security_decision: BLOCK_TESTNET_ASSURANCE_UNRESOLVED_CONCURRENCY_ASSUMPTIONS`

This is expected. The TLA lane models the reviewed client-visible interleavings, but it does not prove backend atomic single-use, remote payload-service serialization internals, raw payload/signature material, or XRPL broadcast/finality behavior.

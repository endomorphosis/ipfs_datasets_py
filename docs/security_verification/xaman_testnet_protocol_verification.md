## Coverage Decision

Tamarin and ProVerif are required.

The approved Testnet model contains remote-payload protocol claims for Testnet network binding, fresh account provenance, review/auth ordering, signing decision ordering, submit UI ordering, and audit redaction boundaries. It also contains explicit gaps for raw payload semantics, raw signature and transaction blob material, XRPL broadcast/finality, replay, refusal, cancellation, and expiry.

The report records every unsupported protocol semantic as `NOT_MODELED`. A missing solver, missing ProVerif projection, unexecuted required solver check, timeout, failed solver run, or unsupported result blocks the Testnet assurance verdict.

## Modeled Scope

`xaman_testnet_payload.spthy` models only reviewed categorical Testnet facts:

- a remote payload issued for XRPL Testnet;
- categorical device payload intake;
- review before auth;
- auth before the observed signing decision;
- signing decision before submit UI attempt/result;
- fresh Testnet account boundary;
- audit redaction preservation.

The Tamarin lemmas are `review_requires_testnet_binding`, `review_requires_fresh_testnet_account`, `auth_requires_review`, `signing_decision_requires_review_and_auth`, `submit_ui_requires_signing_decision`, four event-specific audit-redaction lemmas, and `preserved_attack_traces_are_reachable`. The ProVerif projection independently checks the categorical review, authorization, signing-decision, and submit-result correspondence ordering.

## Not Modeled

The following protocol semantics remain blockers and are not converted into proof claims:

- raw payload JSON, request bodies, transaction intent semantics, and digest-to-signed-byte reconstruction;
- native cryptographic signing output, raw signatures, signed transaction blobs, and account identifiers;
- production runtime equivalence and endpoint authenticity beyond the reviewed Testnet verifier evidence;
- replay, duplicate submission, backend atomic single-use, expiration enforcement, and conflict handling;
- refusal/decline, cancellation, and expiry runtime paths;
- XRPL broadcast request material, mempool behavior, validated-ledger inclusion, and finality.

## Attack Trace Retention

The report must preserve attack traces. It retains the Testnet disproof vectors with `UNKNOWN` status and the fuzz counterexample paths, including replay/duplicate resolution, wrong network, imported production account, raw payload material, raw signature blob, signing before auth, forged broadcast/finality, submit result removal, decline, cancel, and expiry traces.

Those traces remain blocking counterevidence because their associated runtime and backend semantics are not modeled by these bounded protocol projections.
